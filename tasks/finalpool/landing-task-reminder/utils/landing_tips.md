# 公司统一入职任务清单（PDF 素材） — Markdown 版

来源：当前 `landing_task.json`（四个部门 + 公共任务）

## 适用与使用
- 适用范围：所有新入职员工（Backend/Frontend/Testing/Data）。
- 使用方式：打印或导出为 PDF，供 HR/部门经理统一发放与跟踪。
- 完成口径：完成培训/阅读/环境配置，并在系统/表单中提交记录或截图留痕。

## 公共任务（所有部门通用，入职后 7 天内完成）
- Onboarding Training（入职培训）— D+7 内完成。
- Security Training（安全培训）— D+7 内完成。
- Confidentiality Training（保密培训）— D+7 内完成。
- Company Culture（公司文化）— D+7 内完成。
- Company Strategy（公司战略）— D+7 内完成。

## Backend（后端组）分组任务（入职后 30 天内完成）
- Backend Development Process（开发流程）— D+30 内完成。
- Backend Development Standards（开发规范）— D+30 内完成。
- Backend Development Environment（开发环境搭建）— D+30 内完成。

## Frontend（前端组）分组任务（入职后 45 天内完成）
- Frontend Development Process（开发流程）— D+45 内完成。
- Frontend Development Standards（开发规范）— D+45 内完成。
- Frontend Development Environment（开发环境搭建）— D+45 内完成。

## Testing（测试组）分组任务（入职后 60 天内完成）
- Testing Development Process（流程与方法）— D+60 内完成。
- Testing Development Standards（测试规范）— D+60 内完成。
- Testing Development Environment（测试环境/工具）— D+60 内完成。

## Data（数据组）分组任务（入职后 75 天内完成）
- Data Development Process（数据开发流程）— D+75 内完成。
- Data Development Standards（数据开发规范）— D+75 内完成。
- Data Development Environment（数据环境/工具）— D+75 内完成。

## 建议流程与留痕
- 公共任务：在学习平台完成课程与测验，下载结业页截图或导出证书并上传至入职系统。
- 分组任务：完成文档阅读/工具安装/环境校验，提交对应清单或截图（例如 IDE/SDK 版本、脚本运行通过结果）。
- 由直属经理或导师进行节点验收（D+7、D+30/45/60/75），HR 汇总归档。

## 备注
- D+N 含义：以入职当日为 D（Day 0），N 为入职后的自然日天数。
- 如涉及外部系统或网络资源，请使用公司账号登录并遵循信息安全要求。
